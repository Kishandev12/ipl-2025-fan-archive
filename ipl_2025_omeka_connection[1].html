Project Architecture

This IPL 2025 Fan Archive operates as a two-tier digital heritage system designed for academic presentation and archival sustainability:

Frontend Layer (This Site)

Function: Interpretive interface for public engagement
Technology: Static HTML with embedded metadata
Purpose: Narrative-driven exploration of fan content
Audience: Classroom presentation, general users

Backend Layer (Omeka)

Function: Archival repository with standards compliance
Technology: Omeka CMS with Dublin Core metadata
Purpose: Long-term preservation and scholarly access
Audience: Researchers, archivists, future scholars

Why This Dual Approach?

The HTML front-end provides immediate, visually engaging access for classroom demonstrations, while the Omeka backend ensures archival integrity, searchability, and long-term preservation aligned with digital humanities best practices.

Dublin Core Metadata Implementation

Every item in this archive includes Dublin Core metadata fields that map directly to Omeka's data model. This ensures seamless migration from the presentation layer to the archival backend.

Standard Metadata Fields Used

dc:creator Author/Publisher (e.g., ESPNcricinfo, Reddit Community)
dc:date Publication date (ISO 8601 format: YYYY-MM-DD)
dc:type Content type (News Article, Forum Thread, Meme, etc.)
dc:format Media format (Digital Text, Image, Web Page)
dc:subject Keywords and topics (IPL; Memes; Fan Culture)
dc:description Content summary and context
dc:rights Copyright and licensing information
dc:identifier Unique identifier (URL or custom ID)
Omeka Compatibility Note

These Dublin Core fields are native to Omeka's core metadata schema. Each HTML item can be imported directly into Omeka using the Omeka CSV Import plugin or manual entry, with metadata pre-structured for seamless integration.

Archival Workflow: HTML to Omeka

Here's how this interpretive front-end connects to your Omeka archival backend:

1
Content Aggregation (HTML Archive)
This HTML site serves as the first-pass interpretive layer. It aggregates fan content from platforms like Reddit, Twitter/X, and news sites with descriptive narratives and context.
2
Metadata Extraction
Each archive item includes embedded Dublin Core metadata. These can be extracted and formatted as CSV or JSON for bulk import into Omeka, or entered manually through Omeka's admin interface.
3
File Capture & Storage
Screenshots, archived web pages (using Wayback Machine or Archive.today), and downloaded media files are uploaded to Omeka as Item Files, linked to their corresponding metadata records. Critical: Always include both the live URL and an archived URL (from web.archive.org or archive.today) in the metadata to protect against link rot.
4
Omeka Item Creation
In Omeka, each piece of content becomes an Item with full Dublin Core metadata. Items can be organized into Collections (e.g., "IPL 2025 Headlines," "Reddit Discussions," "Memes & Reactions").
5
Exhibit Creation (Optional)
Use Omeka's Exhibit Builder plugin to create narrative exhibits that mirror this HTML archive's structure—showing items in curated sequences with scholarly commentary, similar to a museum exhibit.
6
Public Access & Scholarship
Your Omeka instance becomes a permanent, searchable archive with OAI-PMH compliance for harvesting, persistent URLs, and standards-based metadata for long-term scholarly access.

Web Archiving Strategy

Why Web Archives Matter

The average lifespan of a web page is less than 100 days. News articles get updated, Reddit threads are deleted, tweets disappear, and entire websites go offline. For a digital heritage project like this IPL 2025 archive, link rot is an existential threat.

Archival Services Used

1
Internet Archive Wayback Machine
URL: web.archive.org
Method: Automated crawling + user-initiated "Save Page Now"
Strengths: 1 trillion+ pages archived, institutional trust, open access, temporal snapshots
Use Case: Primary archival infrastructure for all URLs in this project
2
Archive.today / Archive.ph
URL: archive.today / archive.ph
Method: User-initiated instant snapshot
Strengths: Fast, permanent URLs, full-page screenshots, no robots.txt restrictions
Use Case: Backup archive when Wayback Machine misses content

How This Archive Uses Web Archiving

Every URL in the main archive has been preserved through at least one archival service:

  • Live Links: Point to original sources (ESPNcricinfo, Reddit, etc.)
  • Wayback Snapshots: Archived versions accessible via web.archive.org
  • Archive.today Snapshots: Alternative archived versions for redundancy
  • Multiple Captures: Key moments (suspension announcement, final) have snapshots from multiple dates

Creating Your Own Archives

1
Manual Wayback Machine Archiving
Visit web.archive.org/save/[YOUR-URL] to manually save any page. The Wayback Machine will crawl and archive it within minutes. Check the calendar view to see all saved snapshots.
2
Archive.today Instant Snapshot
Go to archive.today, paste your URL, and click "Save". You'll get a permanent URL (e.g., archive.ph/AbC12) that never changes. Great for capturing pages that might be taken down quickly.
3
Document Both URLs in Omeka
In Omeka metadata, include both the original URL (dc:identifier) and the archived URL (dc:relation or dc:source). This ensures researchers can access content even if the original disappears.
Best Practice: The 3-2-1 Rule

For critical content, follow the archival 3-2-1 rule:

3 copies of the content (original + 2 archives)
2 different archival services (Wayback + Archive.today)
1 local backup (screenshot or downloaded HTML)

Integration with Omeka

When creating Omeka items, use Dublin Core fields to document archival status:

dc:identifier https://espncricinfo.com/story/ipl-2025-suspended... (original URL)
dc:relation https://web.archive.org/web/20250509.../espncricinfo... (Wayback)
dc:source https://archive.ph/AbC12 (Archive.today)
dc:description Archived on 2025-05-09 via Wayback Machine and Archive.today

This multi-URL approach ensures that even if ESPNcricinfo takes down the article, researchers can still access it through two independent archives.
